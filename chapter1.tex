\subsection{Анализ программных инструментов разработки}

Для реализации проекта рассматриваются современные полнофункциональные (Fullstack)
подходы. Выбор инструментов обусловлен требованиями к производительности и
масштабируемости:

\begin{itemize}
	\item \textbf{Frontend-фреймворк:} Сравнение React и Vue.js показало преимущество
	      React в части экосистемы библиотек для работы с картами (напр., \texttt{react-leaflet}).
	\item \textbf{Backend-платформа:} Использование среды \textit{Node.js} позволяет о
	      рганизовать асинхронную обработку запросов, что критично при расчете маршрутов.
	\item \textbf{Геоданные:} Использование формата \texttt{GeoJSON} для передачи
	      координат между сервером и клиентом.
\end{itemize}

Для реализации клиентской части приложения выбрана библиотека React. В процессе 
настройки интерфейсов и компонентов использовались официальные руководства по 
быстрому старту из документации разработчиков~\cite{react_docs}.

Для реализации серверной части приложения выбрана платформа \textit{Node.js}, 
архитектура которой подробно описана в работе~\cite{cantelon2017node}. 
Использование неблокирующего ввода-вывода позволяет эффективно 
обрабатывать запросы от множества курьеров одновременно.

\subsection{Формулировка цели и задач работы}

На основе проведенного анализа инструментов (см. подраздел 1.2) и выявленных
пробелов в существующих продуктах, была сформулирована цель исследования.

\textbf{Цель работы} --- создание веб-приложения, обеспечивающего построение
оптимальных маршрутов с использованием алгоритмов на графах.


Для достижения поставленной цели необходимо решить следующие \textbf{задачи}:
\begin{enumerate}
	\item Провести сравнительный анализ современных алгоритмов решения \linebreak транспортных задач и ГИС-технологий;
	\item Разработать архитектуру клиент-серверного приложения и спроектировать структуру базы данных для хранения геоданных;
	\item Реализовать программные модули для управления штатом курьеров и мониторинга статусов доставки;
	\item Интегрировать картографические сервисы и внедрить алгоритм оптимизации маршрутов;
	\item Провести тестирование разработанной системы в условиях имитации реальной нагрузки.
\end{enumerate}
